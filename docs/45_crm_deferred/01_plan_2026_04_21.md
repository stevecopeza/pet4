## Item 1 — Lead: expose `opportunity_id` in domain + API

### Current state (verified 2026-04-22)

- `wp_pet_leads.opportunity_id VARCHAR(36) NULL` — **column exists in DB**.
- `Lead` entity (`src/Domain/Commercial/Entity/Lead.php`) — **no `$opportunityId` property,
  no getter, no `linkOpportunity()`, no `qualify()` method**.
- `SqlLeadRepository::hydrate()` — **does not read `opportunity_id` from the row**.
- `SqlLeadRepository::save()` — **does not include `opportunity_id` in the `$data` array**.
- `LeadController::serializeLead()` — **does not include `opportunityId` in the response**.
  Returns: `id`, `customerId`, `customerName`, `subject`, `description`, `status`,
  `source`, `estimatedValue`, `malleableData`, `createdAt`, `updatedAt`, `convertedAt`.

### What to build

**1a. `Lead` entity** (`src/Domain/Commercial/Entity/Lead.php`)

Add to constructor (after existing params, before `$createdAt`):
```php
private ?string $opportunityId = null
```

Add getter:
```php
public function opportunityId(): ?string { return $this->opportunityId; }
```

Add mutation methods:
```php
public function linkOpportunity(string $opportunityId): void {
    $this->opportunityId = $opportunityId;
}

public function qualify(): void {
    if ($this->status === 'new') {
        $this->status = 'qualified';
    }
}
```

**1b. `SqlLeadRepository::hydrate()`** (`src/Infrastructure/Persistence/Repository/SqlLeadRepository.php`)

Pass `$row->opportunity_id ?? null` as the new constructor parameter.

**1c. `SqlLeadRepository::save()`**

Add to `$data` array:
```php
'opportunity_id' => $lead->opportunityId(),
```
Add `'%s'` to `$format` array.

**1d. `LeadController::serializeLead()`** (`src/UI/Rest/Controller/LeadController.php`)

Add to the returned array:
```php
'opportunityId' => $lead->opportunityId(),
```

### No migration needed — column already exists.

## Item 2 — Lead status + back-link when Opportunity created

### Current state (verified 2026-04-22)

- `CreateOpportunityHandler` (`src/Application/Commercial/Command/CreateOpportunityHandler.php`)
  creates the Opportunity and saves it — **does not touch the Lead**. Constructor injects
  only `TransactionManager` and `OpportunityRepository`.
- Lead's `status` stays as-is (`new` or whatever) after an Opportunity is created from it.
- `wp_pet_leads.opportunity_id` is never written.

### What to build

**2a. `CreateOpportunityHandler`** — inject `LeadRepository`, write back after save:

```php
public function __construct(
    private TransactionManager   $transactionManager,
    private OpportunityRepository $opportunityRepository,
    private LeadRepository        $leadRepository,   // ADD
) {}
```

After `$this->opportunityRepository->save($opportunity)`:

```php
if ($command->leadId()) {
    $lead = $this->leadRepository->findById($command->leadId());
    if ($lead !== null) {
        $lead->linkOpportunity($opportunity->id());
        $lead->qualify();
        $this->leadRepository->save($lead);
    }
}
```

PHP-DI autowires `LeadRepository` automatically — no ContainerFactory change needed.
