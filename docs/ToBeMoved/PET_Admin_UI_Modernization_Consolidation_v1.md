# PET Admin UI Modernization — Consolidation v1

## Purpose
This phase consolidates the modernization work already completed. Focus is on consistency, removal of remaining legacy patterns, and strengthening shared UI primitives.

## Scope
- Remove remaining alert()/confirm()
- Normalize state handling (LoadingState, ErrorState, EmptyState)
- Reduce inline style usage
- Strengthen shared primitives (Dialog, Toast, DataTable)
- Improve accessibility (focus management, keyboard nav)
- Expand regression coverage

## Non-goals
- No API changes
- No domain changes
- No workflow changes

## Output
A fully consistent admin UI using the established foundation primitives.
