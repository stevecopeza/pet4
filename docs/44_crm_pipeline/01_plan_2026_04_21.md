## Overview

The PET plugin currently supports a minimal Lead → Quote path:
`Lead (new/qualified) → [ConvertLeadToQuoteCommand] → Quote (draft)`.

This works, but misses the qualification and pipeline stages that MSP sales teams actually use.
There is no `Opportunity` entity, no qualification checklist, and no pipeline view — just a flat
list of leads and a flat list of quotes.

This sprint introduces a proper **CRM pipeline**: Lead → Opportunity → Quote, with qualification
data, structured stages, and a pipeline Kanban/list view in the portal and admin UI.

---

## Current State

```
Lead ──[convert]──► Quote ──[approve]──► Sent ──[accept/reject]──► Contract/Project
 │
 └── status: new | qualified | converted | lost
```

**Gaps:**
- No intermediate Opportunity stage between Lead and Quote.
- No qualification record (budget, timeline, decision-maker, competition).
- Lead status `qualified` is set manually with no structured criteria.
- No pipeline view — sales team cannot see deals by stage.
- No win-rate or pipeline-value reporting.

---

## Target State

```
Lead ──[qualify]──► Opportunity ──[quote]──► Quote ──[approve/send/accept]──► Delivery
 │                      │
 └── status: new|       └── stage: discovery | proposal | negotiation | closed_won | closed_lost
     qualified|lost          qualification: budget, timeline, decision_maker, competition_notes
```

## Domain Design

### New Entity: `Opportunity`

```php
// src/Domain/Commercial/Entity/Opportunity.php
class Opportunity {
    private string $id;           // UUID
    private int $customerId;
    private ?int $leadId;         // nullable — opportunities can exist without a lead
    private string $name;
    private string $stage;        // discovery | proposal | negotiation | closed_won | closed_lost
    private float $estimatedValue;
    private ?string $currency;
    private ?DateTimeImmutable $expectedCloseDate;
    private int $ownerId;         // WP user ID — the account manager
    private array $qualification; // JSON blob: budget, timeline, decision_maker, competition
    private ?string $notes;
    private DateTimeImmutable $createdAt;
    private ?DateTimeImmutable $updatedAt;
    private ?DateTimeImmutable $closedAt;
    private ?int $quoteId;        // set when opportunity is converted to a quote
}
```

**Stages:**

| Stage | Meaning |
|---|---|
| `discovery` | Initial scoping, understanding requirements |
| `proposal` | Quote being prepared or already sent |
| `negotiation` | Customer reviewing, price/scope discussions |
| `closed_won` | Quote accepted, delivery underway |
| `closed_lost` | Deal lost — reason captured in notes |

**Qualification fields (stored as JSON in `qualification_json`):**

| Field | Type | Description |
|---|---|---|
| `budget_confirmed` | bool | Customer has confirmed budget |
| `budget_amount` | float | Stated budget |
| `timeline` | string | "Q3 2026" or specific date |
| `decision_maker` | string | Name/role of economic buyer |
| `competition` | string | Known competitors in the deal |
| `pain_point` | string | Primary driver / problem being solved |
| `next_step` | string | Agreed next action |

## Database

### New table: `wp_pet_opportunities`

```sql
CREATE TABLE wp_pet_opportunities (
    id            VARCHAR(36)     NOT NULL PRIMARY KEY,
    customer_id   BIGINT UNSIGNED NOT NULL,
    lead_id       BIGINT UNSIGNED NULL,
    name          VARCHAR(255)    NOT NULL,
    stage         VARCHAR(50)     NOT NULL DEFAULT 'discovery',
    estimated_value DECIMAL(12,2) NOT NULL DEFAULT 0,
    currency      VARCHAR(3)      NULL DEFAULT 'ZAR',
    expected_close_date DATE      NULL,
    owner_id      BIGINT UNSIGNED NOT NULL,
    qualification_json LONGTEXT   NULL,
    notes         TEXT            NULL,
    created_at    DATETIME        NOT NULL,
    updated_at    DATETIME        NULL,
    closed_at     DATETIME        NULL,
    quote_id      BIGINT UNSIGNED NULL,
    INDEX idx_customer (customer_id),
    INDEX idx_lead (lead_id),
    INDEX idx_stage (stage),
    INDEX idx_owner (owner_id)
);
```

### Alter `wp_pet_quotes`
Add `opportunity_id VARCHAR(36) NULL` — quotes already have `lead_id`; add `opportunity_id` as
the preferred linkage going forward. Both are nullable; legacy quotes have neither.

### Alter `wp_pet_leads`
Add `opportunity_id VARCHAR(36) NULL` — so a Lead can reference the Opportunity it spawned.

## Application Layer

### Commands

| Command | Handler | Description |
|---|---|---|
| `CreateOpportunityCommand` | `CreateOpportunityHandler` | Open new opportunity (from lead or standalone) |
| `UpdateOpportunityCommand` | `UpdateOpportunityHandler` | Update stage, qualification, value, owner |
| `ConvertOpportunityToQuoteCommand` | `ConvertOpportunityToQuoteHandler` | Creates a Quote in DRAFT, links back |
| `CloseOpportunityCommand` | `CloseOpportunityHandler` | Close won/lost; stamps `closed_at` |

### Convert Lead → Opportunity
`ConvertLeadToOpportunityHandler` — mirrors existing `ConvertLeadToQuoteHandler` but creates an
Opportunity instead of jumping straight to a Quote. The lead transitions to `qualified`.

The existing `ConvertLeadToQuoteCommand` is preserved for backward-compat (older code paths still work),
but new UI will go Lead → Opportunity → Quote.

## REST API

New routes under `pet/v1`:

```
GET    /opportunities                    list (filter by stage, customer, owner)
POST   /opportunities                    create
GET    /opportunities/{id}               get
PATCH  /opportunities/{id}               update stage / qualification / value
POST   /opportunities/{id}/convert-quote create quote from opportunity
POST   /opportunities/{id}/close         close won/lost
```

## Admin UI

### New page: `OpportunitiesPage` (tab in the Leads/Commercial section)

**Pipeline View** (default): Kanban columns — one per stage. Each card shows:
- Opportunity name
- Customer name
- Estimated value
- Expected close date (colour-coded: green = future, amber = <30 days, red = overdue)
- Owner initials

**List View** (toggle): Sortable table with all columns + filter by stage/owner/customer.

**Opportunity detail drawer**: Opens on click. Shows full qualification checklist, notes, linked
lead and quote (with links). "Convert to Quote" button creates a quote and redirects.

### Modify `LeadsPage`
- "Convert" action on a Lead now opens "Create Opportunity" form (pre-filled from lead data)
  rather than jumping straight to a blank quote.
- Show linked Opportunity badge on each lead row (if one exists).

---

## Staff Portal

### New page: `PipelinePage` (`#pipeline` route)

Visible to users with `pet_sales` or `pet_manager`. Shows a read-only pipeline view:
- Kanban by stage, same card format as admin.
- Filter by "My Opportunities" (owned by current user) vs All.
- Clicking a card shows the qualification summary + next step.
